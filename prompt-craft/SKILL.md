# Prompt Craft (提示词修炼)

提示词的本质是**表达**。不是为了润色说明文，而是为了点亮模型向量空间中的特定坐标。

**核心原则：** 得其意而忘其言。少描述，多定义。追求语义压缩，避免注意力漂移。

## Philosophy Layer (道：底层逻辑)

- **本质是表达**：本意（脑中想法）→ 文意（提示词符号）→ 解意（模型解压缩）。
- **效果函数**：`Output = f(模型能力, 任务理解, Prompt质量)`。任务理解（Know-how）是上限，技巧只是逼近上限。
- **从描述走向定义**：描述是“云”，导致偏差；定义是“箭”，直击坐标。消除语义空洞，精准点亮向量空间。
- **Read in, Prompt out**：高质量输出源于深度阅读与理解，而非技巧堆砌。

## Compression Layer (压缩：脱水修炼)

- **脱水路径**：段落 → 句子 → 词语 → 单字。每一步都倒逼自己思考事物的原子核。
- **语义压缩包**：角色名（如“哲学家”）自带知识域、风格与决策倾向。
- **极致凝练**：当核心词汇极少时，Attention 机制没有漂移空间，泛化能力最强。

## Structure Layer (结构：骨架与形)

- **Markdown (结构化)**：利用 `# Role`, `## Rules`, `## Workflow` 实现语义聚拢，适合复杂逻辑协作。
- **Lisp (语义压缩)**：抛弃修饰，直接定义函数与逻辑，追求极致精准（点亮星星）。
- **混合模式**：以结构化为“形”（稳定接口），以压缩词为“意”（模块内部高密度投喂）。

## Attention Layer (注意力控制)

- **消除空洞**：删掉对输出无实质贡献的“片汤话”。
- **语义聚拢**：利用标识符（#, -, XML/JSON 闭合）聚拢同类语义，降低认知负担。
- **任务拆解**：复杂任务构建 Prompt Chain，避免单个 Prompt 权重过载。

## Role Layer (角色：语义锚点)

- **角色是压缩包**：分配角色即调用模型预训练的知识簇。
- **解压缩维度**：若模型认知存在盲区，需注入 `Background`, `Skills`, `Rules`, `Constraints`, `Workflow` 进行解压。

## Golden Words Layer (金阶技巧)

- **引导策略**：正面引导（用“简洁”不用“不要太复杂”），减少否定词。
- **语言习惯**：使用常规用词，避免生僻词干扰。
- **思维加速**：`take a deep breath`, `think step by step`, `Step-Back` (后退一步策略)。
- **Few-shots (示例)**：代表性强、难度递进、格式统一。

## Format Layer (格式适配)

- **GPT**：偏好 YAML / Markdown。
- **Claude**：偏好 XML 标签（`<context>`, `<instruction>`）。
- **Gemini**：偏好 JSON 结构。
- **布局顺序**：重要信息（角色、目标）前置，参考信息（背景、资料）后置。

## Workflow

1. **理解本质 (本意)**：明确任务的底层逻辑与方法论。
2. **选择适配 (形)**：根据模型偏好与复杂度选择格式（Markdown/XML/Lisp）。
3. **分配压缩包 (核)**：选定核心角色，必要时进行维度解压。
4. **构建骨架 (形)**：建立结构化框架，聚拢语义块。
5. **填充核心词 (意)**：填入高信息密度的定义词，替换模糊描述。
6. **压缩脱水 (减薄)**：删除冗余，确保 Attention 权重集中。
7. **检查验证 (证)**：验证注意力分布，消除逻辑矛盾。

## When to Use / Not Use

- **使用场景**：从零构建复杂 Agent、需要极高执行稳定性、核心逻辑需要深度泛化、生产级应用。
- **不使用场景**：已有 Prompt 的局部修补（用 `prompt-refactor`）、简单一次性问答、纯文案润色。

## Common Mistakes

- **垃圾进垃圾出**：自己没想透（无 Know-how），试图靠 Prompt 技巧让 AI 变聪明。
- **注意力漂移**：提示词过长且缺乏结构，导致模型忽略核心规则。
- **过度解释**：写了一堆“为什么”，却没写清“怎么做”（执行指令不足）。

## Output Standard

- 核心意图是否通过高密度词汇（压缩包）表达？
- 结构是否清晰（语义聚拢）？
- 是否消除了否定词与语义空洞？
- 格式是否适配目标模型？
