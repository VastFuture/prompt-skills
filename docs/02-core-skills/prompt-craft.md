# prompt-craft — 提示词修炼

> **一句话描述：** 从零构建高质量、生产级提示词。

---

## 核心理念

> "得其意而忘其言。少描述，多定义。"

**核心原则：** 追求语义压缩，避免注意力漂移。提示词的本质是**表达**，不是为了润色说明文，而是为了点亮模型向量空间中的特定坐标。

---

## 七层修炼框架

### 1️⃣ Philosophy Layer（本意层）

本意（脑中想法）→ 文意（提示词符号）→ 解意（模型解压缩）

- 效果函数：`Output = f(模型能力, 任务理解, Prompt质量)`
- 任务理解（Know-how）是上限，技巧只是逼近上限
- 从描述走向定义：描述是"云"，导致偏差；定义是"箭"，直击坐标

### 2️⃣ Compression Layer（压缩层）

段落 → 句子 → 词语 → 单字，压缩至原子

- 每一步都倒逼自己思考事物的原子核
- 当核心词汇极少时，Attention 机制没有漂移空间
- 语义压缩包：角色名（如"哲学家"）自带知识域、风格与决策倾向

### 3️⃣ Structure Layer（结构层）

三种模式可选：

- **Markdown（结构化）**：适合复杂逻辑协作，用 `# Role`、`## Rules`、`## Workflow` 实现语义聚拢
- **Lisp（语义压缩）**：抛弃修饰，直接定义函数与逻辑，追求极致精准
- **混合模式**：以结构化为"形"（稳定接口），以压缩词为"意"（模块内部高密度投喂）

### 4️⃣ Attention Layer（注意力控制）

- **消除空洞**：删掉对输出无实质贡献的"片汤话"
- **语义聚拢**：利用标识符（#、-、XML/JSON 闭合）聚拢同类语义
- **任务拆解**：复杂任务构建 Prompt Chain，避免单个 Prompt 权重过载

### 5️⃣ Role Layer（角色层）

角色 = 压缩包

- 分配角色即调用模型预训练的知识簇
- 若模型认知存在盲区，需注入 `Background`、`Skills`、`Rules`、`Constraints`、`Workflow` 进行解压

### 6️⃣ Golden Words Layer（金阶技巧）

- **正面引导**：用"简洁"不用"不要太复杂"，减少否定词
- **Few-shots（示例）**：代表性强、难度递进、格式统一
- **思维加速**：使用 `take a deep breath`、`think step by step`、`Step-Back`（后退一步策略）

### 7️⃣ Format Layer（格式适配）

根据模型偏好选择格式：

| 模型 | 偏好格式 |
|------|----------|
| GPT | YAML / Markdown |
| Claude | XML 标签（`<context>`, `<instruction>`）|
| Gemini | JSON 结构 |

## 标准工作流

```
理解本质 → 选择适配 → 分配压缩包 → 构建骨架 → 填充核心词 → 压缩脱水 → 检查验证
   ↓           ↓           ↓           ↓           ↓           ↓           ↓
  本意        形          核          形          意          减薄        证
```

**7 步详解：**

1. **理解本质（本意）**：明确任务的底层逻辑与方法论
2. **选择适配（形）**：根据模型偏好与复杂度选择格式
3. **分配压缩包（核）**：选定核心角色，必要时进行维度解压
4. **构建骨架（形）**：建立结构化框架，聚拢语义块
5. **填充核心词（意）**：填入高信息密度的定义词
6. **压缩脱水（减薄）**：删除冗余，确保 Attention 权重集中
7. **检查验证（证）**：验证注意力分布，消除逻辑矛盾

---

## 适用场景

- 从零构建复杂 Agent
- 需要极高执行稳定性
- 核心逻辑需要深度泛化
- 生产级应用

## 不适用场景

- 已有 prompt 的局部修补（用 `prompt-refactor`）
- 简单一次性问答
- 纯文案润色

## 实战案例

### 案例 1：客服智能体（Markdown 结构化）

```markdown
# Role: 售后温情管家
## Profile:
- 背景: 负责处理客户的不满与订单追踪
- 风格: [共情倾听], [高效闭环], [专业克制]
## Rules:
- 严禁[模板化]冷冰冰回复
- 必须首先点亮[情绪缓冲]坐标，再进行[事实核对]
- 边界外需求必须引导至[人工介入]
## Workflow:
1. 识别客户情绪并进行[语义镜像]回复
2. 提取订单号，调用`search_order`工具
3. 根据订单状态选择[透明化解释]路径
4. 确认问题解决并进行[满意度预埋]
```

### 案例 2：简单任务降维（Few-shots + CoT）

```markdown
# Role: 擅长[科普降维]的老师
# Goal: 解释"线粒体"

## Few-shots:
- 问: 什么是血液循环？
- 答: 就像城市里的快递小哥，不停地把氧气包裹送到每个家门口。

## Instruction:
请[后退一步思考]: 先分析能量对身体的重要性，再用[工厂/电池]进行比喻。
Take a deep breath and think step by step.
```

## 核心金词参考

| 场景 | 推荐金词 |
|------|----------|
| 思维指令 | `think step by step`、`take a deep breath`、`Step-Back Prompting` |
| 质量提示 | `do it right and I'll give you a tip` |
| 逻辑控制 | `First... Then... Finally...`、`反向PUA: 先列10个反对理由` |
| 精准定位 | "点亮星星"（语义精准指向） |
| 深度分析 | "Read in, Prompt out" |

## 自检清单

改完 prompt 后，至少自查这 6 项：

- [ ] **去水**：是否删除了所有"片汤话"和解释性长句？
- [ ] **定性**：核心意图是否已转化为精准定义词？
- [ ] **聚拢**：是否使用了 #、-、或闭合标签聚拢语义？
- [ ] **去否**：是否将"不要..."转化为了正面指令？
- [ ] **适配**：格式是否符合目标模型的偏好？
- [ ] **深度**：本意是否清晰？是否 Read in 了足够的 Know-how？

---

## 常见误区

| 误区 | 后果 |
|------|------|
| 自己没想透，试图靠技巧让 AI 变聪明 | 垃圾进垃圾出 |
| 提示词过长且缺乏结构 | 注意力漂移 |
| 写了一堆"为什么"，却没写清"怎么做" | 执行指令不足 |
