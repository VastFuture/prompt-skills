# CRISPE 框架提示词优化

> **核心理念**：给 AI 一个完整的专家档案，让它像真正的专业人士一样思考和回答。

---

## 一句话描述

CRISPE 框架通过角色 + 背景 + 目标 + 技能 + 格式 + 工作流的六层结构，把普通提示词升级为专业级专家指令。

---

## CRISPE 框架图解

### C — Capacity / Role
**角色身份**

定义 AI 应该扮演的专家角色。

```
你是一个 [角色名称]
```

**选角色小技巧**：
- 越具体越好："资深前端工程师" > "程序员"
- 可以是职业角色，也可以是特定身份
- 多个角色用"既是...又是..."组合

---

### R — Profile
**背景档案**

说明这个专家的来历、背景、专长。

```
### Profile
- author: [作者]
- version: [版本]
- language: [语言]
- description: [角色描述]
```

**包含要素**：
- 为什么这个"专家"适合回答这个问题？
- 它有什么独特的经验和视角？
- 它的专业边界在哪里？

---

### I — Insight
**洞察注入**

提供任务相关的背景知识、行业洞察。

```
### Insight
- [相关行业知识]
- [关键背景信息]
- [专业判断依据]
```

**作用**：
- 帮 AI 进入"专家思维模式"
- 避免泛泛而谈，提供有深度的回答
- 减少幻觉，提升专业性

---

### S — Statement
**任务声明**

明确要解决的问题和目标。

```
### Goals
- [目标1]
- [目标2]
- ...
```

**写法**：
- 用动词开头（分析、设计、评估、对比）
- 目标要可量化或可验证
- 列出优先级，主目标放前面

---

### P — Personality
**人格设定**

定义回答的风格、语气、个性。

```
### Personality
- [专业严谨] / [轻松幽默] / [直接犀利]
- [喜欢用表格] / [偏好分点陈述]
- [注重数据] / [注重故事性]
```

**为什么重要？**
同样一个问题，不同人格设定的 AI 会给出截然不同的回答风格。

---

### E — Experiment
**探索实验**

引导 AI 主动探索、测试边界、提供多种可能。

```
### Experiment
- 如果信息不足，主动说明需要什么
- 提供多个方案时，说明各自的优缺点
- 在不确定的地方给出置信度
```

**核心理念**：不要让 AI 只给一个答案，要让它像一个负责任的专家一样思考所有可能性。

---

## 使用示例

### 优化前（普通提示词）

```
帮我分析一下这个产品为什么卖不动。
```

### 优化后（CRISPE 结构）

```markdown
# Role: 资深产品增长分析师

## Profile
- author: GrowthLab
- version: 1.0
- language: 简体中文
- description: 专注消费品领域的增长策略专家，擅长通过数据分析和市场洞察识别增长障碍

### Skill
- 熟练使用AARRR模型分析用户生命周期
- 擅长竞品对比分析和差异化定位
- 能够通过用户访谈识别关键痛点
- 熟悉电商转化率优化策略
- 擅长将数据洞察转化为可执行建议

## Goals
- 识别产品销量下滑的核心原因
- 分析用户旅程中的关键流失节点
- 找出3-5个最具影响力的改进方向
- 提供数据支撑的增长建议

## Constrains
- 基于现有数据推导，不过度假设
- 每个结论需要有数据或逻辑支撑
- 建议要具体可执行，避免空泛结论
- 分析过程要透明，逻辑可追溯

## OutputFormat
- 问题诊断总结（300字以内）
- 原因分析（按影响程度排序）
- 改进建议（每个建议说明预期效果）

## Workflow
1. 首先阅读产品数据概况，建立整体认知
2. 然后分析用户行为数据和转化漏斗
3. 接着对比竞品，找出差异化机会
4. 接着识别3-5个关键改进点
5. 最后提供具体可执行的增长建议

## Initialization
你好！我是产品增长分析师。给我看看你的产品数据，我来帮你诊断增长瓶颈。
```

## 与六要素模板的区别

| 对比维度 | 六要素模板 | CRISPE框架 |
|---------|-----------|-----------|
| **适用场景** | 快速写作、结构化输出 | 深度任务、复杂分析 |
| **详细程度** | 简洁明了 | 全面专业 |
| **技能定义** | 无 | 有（Skill模块） |
| **工作流** | 可选 | 必须（5步以上） |
| **学习门槛** | 低 | 中 |
| **输出质量** | 稳定 | 专业级 |

### 什么时候用哪个？

| 场景 | 推荐 |
|------|------|
| 写一封邮件、发一条朋友圈 | 六要素（够用） |
| 产品分析报告、战略建议 | CRISPE（更专业） |
| 快速问答、简单查询 | 两者都不用，直接问 |
| 需要 AI 像专家一样思考 | CRISPE（推荐） |

## 快速检查

- [ ] 角色是否具体且专业？
- [ ] Profile 是否说明了这个角色的背景？
- [ ] Insight 是否提供了有价值的洞察？
- [ ] Goals 是否列出 3-5 个明确目标？
- [ ] Personality 是否定义了回答风格？
- [ ] OutputFormat 是否清晰可执行？
- [ ] Workflow 是否包含 5 步以上的完整流程？
- [ ] Initialization 是否让 AI 正确进入角色？

---

## 常见误区

- **误区 1**：角色写得太泛（如"你是AI"）
- **误区 2**：Goals 和 Constrains 混在一起
- **误区 3**：Workflow 只有 2-3 步，不够详细
- **误区 4**：OutputFormat 描述模糊（如"格式清晰"）

> **提示**：CRISPE 框架的核心价值在于**让 AI 像真正的专家一样工作**，而不是像一个答题机器。
