# 六要素提示词模板

> **核心理念**：给 AI 一个完整的舞台，让它清楚知道自己是谁、为什么做、怎么做。

---

## 一句话描述

六要素结构（Role + Background + Task + Requirements + Example + Reflection）让提示词像一份完整的任务书，AI 拿到就知道怎么执行。

---

## 六要素详解

### 1. Role — 角色身份

赋予 AI 一个明确的专家身份。

```
你是一个 [角色名称]
```

**为什么重要？**
角色决定了 AI 的思维框架和语言风格。告诉 AI "你是一个资深产品经理" vs "你是一个 AI"，输出的质量完全不同。

**常见角色**：
- 资深产品经理、文案专家、数据分析师
- 法律顾问、技术架构师、营销策划专家
- 资深编辑、翻译专家、代码审查员

---

### 2. Background — 任务背景

告诉 AI 任务的来龙去脉，让它理解上下文。

```
我正在 [背景/场景] 下，需要 [目标]
我的受众/用户是 [目标人群]
```

**包含要素**：
- **场景**：在什么情况下提出这个需求？
- **受众**：输出给谁看？
- **目的**：解决什么问题？

---

### 3. Task — 核心动作

明确告诉 AI 要做什么。动词要具体、清晰。

```
我希望执行： [具体动作]
```

**好例子**：
- "分析这段用户评论的情感倾向"
- "将以下产品功能描述转化为面向开发者的技术文档"
- "对比三种数据库方案的优缺点"

**坏例子**：
- "帮我看看"（太模糊）
- "优化一下"（不知道优化什么）

---

### 4. Requirements — 输出约束

这是最关键的精调环节。定义输出的风格、格式、结构。

| 约束维度 | 可选项 |
|---------|-------|
| **风格** | 正式 / 口语化 / 专业严谨 / 轻松幽默 |
| **格式** | 列表 / 表格 / JSON / Markdown / 代码块 |
| **结构** | 引言 + 正文 + 结论 / 分点阐述 |
| **人称** | 第一人称 / 第三人称 |
| **语言** | 中文 / 英文 / 中英混合 |
| **长度** | 500字以内 / 三段式 / 精简版 |
| **引用** | 需要标注来源 / 无需引用 |
| **其他** | 关键词加粗 / 代码高亮 / 禁止某类内容 |

> **提示**：Requirements 是新手最容易忽略的要素，也是输出质量最大的变量。写提示词时，**至少写 3 条约束**。

---

### 5. Example — 示例参考

提供正面或反面示例，帮助 AI 快速理解你的期望。

```
例如： [具体示例]
或：不要这样： [反面示例]
```

**作用**：
- 让 AI 理解你期望的"好"长什么样
- 比抽象描述更有效
- 可以是正例（符合预期的输出）或反例（要避免的输出）

---

### 6. Reflection — 反思自检

引导 AI 对自己的输出进行自我评估和追问。

```
请 [反思操作]
```

**常见用法**：
- "请对你刚才的回答进行评分，并说明扣分原因"
- "请检查是否满足上述所有要求，列出未满足项"
- "如果信息不足，请明确告知我需要补充什么"
- "你对我刚才的提问有什么疑问吗？"

---

## 完整填空模板

```markdown
你是一个 [角色身份]，我正在 [背景/场景] 下，需要你 [核心任务]。
请你按照以下要求完成：
  - 风格： [风格描述]
  - 格式： [格式要求]
  - 长度： [字数/篇幅]
  - 其他： [其他约束]

  这里有一个参考示例：[示例内容]

  最后，请 [反思要求]，确保输出满足要求。
```

## 使用示例

### 场景：写一封商务邮件

```
你是一个 [资深商务秘书]，
我正在 [准备与某重要客户签约] 的背景下，需要你 [撰写一封正式的商务邮件催促进展]。
请你按照以下要求完成：
  - 风格：专业但不刻板，礼貌但有力度
  - 格式：纯文本，一段式
  - 长度：200字以内
  - 语气：主动但不催促，表达合作诚意

  这里有一个参考示例：
  "尊敬的王总，感谢您对本次合作的重视。为了推进项目进程，现特此致函确认相关细节..."

  最后，请 [检查语气是否合适、是否包含关键信息]，确保邮件达到目的。
```

## 什么时候用

### 推荐使用

- 任何新提示词写作的第一步
- 需要结构化输出（报告、邮件、文档）时
- 给 AI 的需求比较复杂、多维度时
- 作为教学工具，帮助理解提示词结构

### 不推荐使用

- 简单的一次性问题（"今天天气怎么样？"不需要六要素）
- 需要 AI 自由发挥的创意任务
- 时间紧张、要求快速响应的场景

---

## 快速检查

在发送提示词前，问自己：

- [ ] 角色是否明确？（Role）
- [ ] 背景是否说清楚了？（Background）
- [ ] 任务动作是否具体？（Task）
- [ ] 至少写了 3 条约束？（Requirements）
- [ ] 有示例辅助理解吗？（Example）
- [ ] 需要 AI 做自我检查吗？（Reflection）

> **TIP**：六要素不是每条都要写满。简单任务可以精简，但结构和思路保持一致。
