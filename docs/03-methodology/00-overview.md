# 方法论技能库

> 提示词工程的方法论层 — 从模板到质检的完整工作流

---

## 使用流程

```
模板型 ──→ 原则型 ──→ 质检型 ──→ 重构
  ④           ③           ②         ①
```

**怎么用？**
1. 选一个**模板**快速起稿
2. 用**原则**打磨优化
3. 送**质检**查漏补缺
4. 发现问题，回到第 1 步**重构**

---

## 技能卡片总览

### 模板型 — 快速起稿

| 技能 | 一句话描述 | 什么时候用 |
|------|-----------|------------|
| [六要素提示词模板](./prompt-template.md) | Role + Background + Task + Requirements + Example + Reflection 六步结构 | 任何新提示词写作，快速构建结构化初稿 |
| [CRISPE 框架](./prompt-role.md) | Capacity + Insight + Statement + Personality + Experiment 五步优化 | 想让提示词更专业、更有深度，适合复杂任务 |

### 原则型 — 理念打磨

| 技能 | 一句话描述 | 什么时候用 |
|------|-----------|------------|
| [提示词原则](./prompt-principles.md) | 通用法则 + Anthropic工程师实践（目标清晰 + 直接沟通 + 迭代 + 诚实） | 理解提示词根本逻辑 + 减少过度设计 |

### 质检型 — 查漏补缺

| 技能 | 一句话描述 | 什么时候用 |
|------|-----------|------------|
| [提示词评分与质检](./prompt-score.md) | 通用10维评分 + O1/O3-mini专属质检，一个技能覆盖所有提示词质检场景 | 想量化评估提示词质量，或检查推理模型提示词 |

### 平台型 — 场景落地

| 技能 | 一句话描述 | 什么时候用 |
|------|-----------|------------|
| [Coze 平台提示词写法](./prompt-coze.md) | Coze 平台上 JSON 变量 + Agent 描述对齐的规范 | 在 Coze 上构建智能体，需要配置输入/输出变量 |
